## 完整图景

```
┌─────────────────────────────────────────────────────────────┐
│  1. 用户与 React 组件交互                                   │
│  2. 组件调用生成的指令构建器                                 │
│  3. 钱包签名，交易发送到 Solana                              │
│  4. Anchor 程序验证并修改账户状态                            │
│  5. 前端轮询更新状态，重新渲染                                │
└─────────────────────────────────────────────────────────────┘
```

这个循环就是整个应用程序。用户点击，客户端构建指令，钱包签名，程序更新状态，UI 刷新。一旦您内化了这个循环，Solana 开发中的其他一切都会变得更容易。

## 关键要点

1. **程序是无状态的** — 账户持有所有状态
2. **PDA 实现无需信任的托管** — 没有私钥持有资金
3. **代码生成消除序列化错误** — 通过构造实现类型安全
4. **IDL 是合约** — 连接链上和链下

如果您只记住一件事，请记住：程序很小，账户是真相，客户端应该是生成的。这种组合使您在项目增长时保持理智。

另外，不要低估保持简单的重要性。状态模型越简单，调试越容易，扩展也越容易。大多数生产错误并不巧妙；它们只是层之间不匹配的假设。
